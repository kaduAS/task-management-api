# Build Summary

This backend API was built as a hands-on project to consolidate my backend knowledge through practical implementation.

The application is a task management API with user authentication and database persistence. The server code is organized inside the `src` directory, the database schema is managed with Prisma, and the project includes documentation files for setup and deployment.

During development, I implemented authentication using JWT, protected routes with middleware, and CRUD operations for tasks. One of the main difficulties was ensuring that authentication logic was correctly applied across the application and that users could only access their own resources.

I also spent time configuring environment variables, database connections, and understanding how the application behaves in both local and deployed environments.

### Key Points

- Implemented authentication and authorization using JWT  
- Built protected CRUD endpoints for task management  
- Used Prisma to manage the PostgreSQL database  
- Structured the project to be readable and maintainable  
- Added setup and deployment instructions  

Next steps for this project include running it locally, pushing it to GitHub, deploying it to a cloud platform, and adding it to my resume as a backend portfolio project.
