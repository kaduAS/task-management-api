# Project Overview

## Purpose

I built this project to practice backend fundamentals by creating a complete task management API from scratch.

The main goal was to understand how core backend components work together in a real application, such as authentication, protected routes, database access, and API structure.

## Implementation

I implemented user registration and login using JWT, along with full CRUD functionality for tasks.  
Each user can only access and modify their own data, which required careful handling of authentication and request validation.

## Challenges

One of the main challenges was organizing the codebase in a clean and scalable way.  
I separated responsibilities between routes, controllers, and middleware, and made the authentication logic reusable across the application.

## Database

I used Prisma to model the database and interact with PostgreSQL.  
This helped me better understand schema design, relationships, and how ORM queries work in practice.

## Summary

This project represents my current understanding of backend development and how I approach building, structuring, and deploying a backend service.
